# Features

This file tracks all features as user stories. Each feature must have:
1. A user story authored by the **Business Analyst**
2. A UX/UI design proposal reviewed and approved by the **UX/UI Designer**
3. Approval from both before development begins

---

## Template

### [FEAT-000] Feature Title

**Status**: `Draft` | `Design` | `Ready` | `In Progress` | `Done`

**User Story**
> As a [type of user], I want [an action], so that [a benefit/value].

**Acceptance Criteria**
- [ ] Criterion 1
- [ ] Criterion 2

**UX/UI Proposal**
> _(UX/UI Designer fills this in. Iterate with Business Analyst until approved.)_

**Notes**
> _(Any additional context, constraints, or open questions.)_
